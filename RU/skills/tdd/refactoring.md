# Кандидаты на рефакторинг

После TDD-цикла ищи:

- **Дублирование** -> выноси в функцию/класс
- **Длинные методы** -> разбивай на приватные хелперы (тесты оставляй на публичном интерфейсе)
- **Поверхностные модули** -> объединяй или углубляй
- **Feature envy** -> переноси логику туда, где живут данные
- **Primitive obsession** -> вводи value objects
- **Проблемные места в существующем коде**, которые подсветил новый код
