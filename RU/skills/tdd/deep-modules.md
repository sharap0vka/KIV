# Глубокие модули

Из книги "A Philosophy of Software Design":

**Глубокий модуль** = маленький интерфейс + много реализации

```
┌─────────────────────┐
│ Небольшой интерфейс │  ← Мало методов, простые параметры
├─────────────────────┤
│                     │
│                     │
│ Глубокая реализация │  ← Сложная логика скрыта внутри
│                     │
│                     │
└─────────────────────┘
```

**Поверхностный модуль** = большой интерфейс + мало реализации (избегать)

```
┌─────────────────────────────────┐
│      Большой интерфейс          │  ← Много методов, сложные параметры
├─────────────────────────────────┤
│   Тонкая реализация             │  ← Просто проксирует дальше
└─────────────────────────────────┘
```

При проектировании интерфейсов спрашивай:

- Можно ли уменьшить число методов?
- Можно ли упростить параметры?
- Можно ли скрыть больше сложности внутри?
