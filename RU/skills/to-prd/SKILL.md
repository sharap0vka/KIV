Этот навык берет текущий контекст диалога и понимание кодовой базы и формирует PRD. НЕ проводи интервью с пользователем - синтезируй то, что уже известно.

## Процесс

1. Изучи репозиторий, чтобы понять текущее состояние кодовой базы (если еще не делал этого).

2. Набросай ключевые модули, которые нужно создать или изменить для реализации. Активно ищи возможности выделить deep modules, которые можно тестировать изолированно.

Deep module (в отличие от shallow module) инкапсулирует много функциональности за простым, тестируемым интерфейсом, который редко меняется.

Сверь с пользователем, что эти модули соответствуют ожиданиям. Уточни, для каких модулей пользователь хочет тесты.

3. Напиши PRD по шаблону ниже и опубликуй его как GitHub issue.

<prd-template>

## Problem Statement

Проблема, с которой сталкивается пользователь, с точки зрения пользователя.

## Solution

Решение проблемы, с точки зрения пользователя.

## User Stories

ДЛИННЫЙ нумерованный список пользовательских историй. Каждая история должна быть в формате:

1. As an <actor>, I want a <feature>, so that <benefit>

<user-story-example>
1. As a mobile bank customer, I want to see balance on my accounts, so that I can make better informed decisions about my spending
</user-story-example>

Этот список должен быть максимально полным и покрывать все аспекты фичи.

## Implementation Decisions

Список принятых решений по реализации. Может включать:

- Какие модули будут созданы/изменены
- Какие интерфейсы модулей будут изменены
- Технические уточнения от разработчика
- Архитектурные решения
- Изменения схем
- API-контракты
- Конкретные взаимодействия

НЕ включай конкретные пути к файлам или фрагменты кода - они быстро устаревают.

## Testing Decisions

Список принятых решений по тестированию. Включи:

- Что делает тест хорошим (проверяет внешнее поведение, а не детали реализации)
- Какие модули будут тестироваться
- Примеры и прецеденты для тестов (похожие тесты в кодовой базе)

## Out of Scope

Описание того, что не входит в рамки этого PRD.

## Further Notes

Любые дополнительные заметки по фиче.

</prd-template>
